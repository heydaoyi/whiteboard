# 无限画布白板软件研发方案

## 一、 项目概述
本项目旨在开发一款基于 B/S (Browser/Server) 架构的在线白板软件。核心特性包括无限画布、矢量级无损缩放、丰富的笔触效果（铅笔、钢笔、水墨等）、多项目管理、云端保存与本地下载，并重点针对移动端（如安卓平板浏览器）进行交互适配。

## 二、 技术选型

### 1. 前端技术栈
*   **核心框架**: Vue 3 或 React (推荐 React，生态中白板相关库较多)
*   **图形渲染引擎**: 
    *   **方案 A (推荐)**: `Konva.js` 或 `Paper.js`。`Paper.js` 在矢量路径处理和贝塞尔曲线平滑方面表现优异，非常适合做笔锋效果。
    *   **方案 B**: 基于 `tldraw` 或 `Excalidraw` 的底层引擎进行二次开发。
*   **笔触算法**: `perfect-freehand` (用于根据鼠标/触摸的移动速度和压力生成具有粗细变化的完美手绘多边形路径，极大地增强水墨、钢笔的真实感)。
*   **状态管理**: Zustand 或 Pinia，用于管理画布状态、工具选择、项目列表等。
*   **构建工具**: Vite

### 2. 后端技术栈
*   **运行环境**: Node.js
*   **Web 框架**: Express 或 Koa (轻量级，易于快速搭建 RESTful API)
*   **数据存储**: 直接使用文件系统 (File System，`fs` 模块)。白板数据序列化为 JSON 格式，保存在服务端程序同级的 `projects` 目录下。

## 三、 核心功能设计思路

### 1. 无限画布与矢量缩放
*   **实现原理**: 采用“摄像机 (Camera)”模型。前端实际渲染的 Canvas 尺寸固定为浏览器窗口大小。通过维护一个全局的 `transform` 矩阵（包含 `x`, `y` 偏移量和 `scale` 缩放比例），将用户的屏幕坐标映射到虚拟的无限世界坐标中。
*   **矢量特性**: 所有绘制的线条不保存为像素，而是保存为一系列的点坐标和路径参数。缩放时，重绘这些矢量路径，从而实现无限放大不失真。

### 2. 多种笔触与粗细调节
*   **数据采集**: 监听 `pointerdown`, `pointermove`, `pointerup` 事件。获取坐标 `(x, y)` 以及压力值 `pressure` (如果设备支持，如 Apple Pencil 或部分安卓平板触控笔)。
*   **笔触模拟**:
    *   **铅笔**: 恒定宽度的线条，可稍微添加噪点或透明度。
    *   **钢笔**: 宽度随绘制速度和压力动态变化，起笔和收笔处较尖锐。
    *   **水墨**: 结合 `perfect-freehand` 生成带有水墨晕染边缘的多边形，并配合半透明的黑色/灰色填充。
*   **粗细调节**: 暴露基础宽度参数 `baseWidth`，结合压力系数计算最终渲染宽度。

### 3. 移动端与平板适配 (安卓平板浏览器)
*   **事件统一**: 统一使用 Pointer Events API (`pointerdown`, `pointermove` 等) 来兼容鼠标、触摸和触控笔。
*   **手势识别**:
    *   **单指/触控笔**: 绘制操作。
    *   **双指**: 拦截默认的页面滚动和缩放行为 (`e.preventDefault()`)。计算双指中心点进行画布平移 (Pan)，计算双指距离变化进行画布缩放 (Pinch-to-zoom)。
*   **防误触**: 识别 `pointerType`，如果支持触控笔，可开启“仅触控笔绘制”模式，此时手指触摸仅用于平移和缩放画布。
*   **UI 适配**: 工具栏采用悬浮面板或侧边栏设计，图标尺寸需符合触控标准（至少 44x44 px）。

### 4. 项目管理与存储 (保存与下载)
*   **数据结构**: 
    ```json
    {
      "projectId": "xxx",
      "version": "1.0",
      "camera": { "x": 0, "y": 0, "scale": 1 },
      "elements": [
        { "id": "1", "type": "path", "tool": "ink", "points": [...], "color": "#000", "width": 5 }
      ]
    }
    ```
*   **服务端保存**: 前端将上述 JSON 数据通过 POST 请求发送至后端。后端在同级目录 `projects/` 下以 `projectId.json` 命名保存文件。
*   **项目切换**: 后端提供 GET `/api/projects` 接口，读取 `projects/` 目录下的所有 JSON 文件名并返回。前端展示列表供用户选择加载。
*   **本地下载**: 前端将当前画布数据转换为 Blob 对象，生成 Object URL，通过创建隐藏的 `<a>` 标签触发浏览器下载，格式可为 `.json` (工程文件) 或导出为 `.png` / `.svg` (图片文件)。

## 四、 研发步骤规划

1.  **阶段一：基础设施搭建 (1周)**
    *   搭建前后端项目脚手架。
    *   实现后端的项目列表读取、保存 JSON 文件接口。
    *   前端实现基础的 Canvas 渲染环境和摄像机缩放/平移逻辑。
2.  **阶段二：核心绘制功能 (2周)**
    *   引入 `perfect-freehand`，实现基础的线条绘制。
    *   开发铅笔、钢笔、水墨三种不同笔刷的渲染逻辑。
    *   实现颜色、粗细切换功能。
3.  **阶段三：移动端适配与手势 (1周)**
    *   完善 Pointer Events 处理，加入双指缩放和平移算法。
    *   在安卓平板浏览器上进行真机调试，解决默认手势冲突和性能卡顿问题。
4.  **阶段四：业务逻辑与联调 (1周)**
    *   实现项目列表展示、新建项目、切换项目。
    *   实现云端保存和本地下载功能。
    *   UI 细节打磨和整体性能优化。
